moo-cdc - AVR firmware-based USB to serial converter

This is an improved software AVR-based usb to serial converter based on www.recursion.jp/avrcdc/.


Serial converter using moo-cdc

It uses V-USB firmware-based USB protocol stack for AVR.

Some tricks/hacks have been made to achieve stable communication at 115200 baud:

  • 256-byte serial RX buffer placed on 0x??00 aligned address boundary
  • RX buffer data pointer indexed by single dedicated register
  • stackable RX interrupts
  • assembly-optimised RX interrupt routine
  • RX interrupt routine works with AVRs with serial registers outside of sbi/cbi instruction reach (Atmega88)
  • 8-byte serial TX buffer
  • stores 1-8 bytes from USB packet, disables data receiving until all data sent through serial
  • supported bauds 9600, 19200, 38400, 57600, 115200
  • parity/stop bit setup
  • parameter setting works with both Windows and Linux
  • to be used with 16Mhz AVR clock
  • LED indicating received data

I am using this converter mainly for console and X-MODEM firmware upload. The only problems encountered are those ralated to baudrate mismatch - AVR running at 16Mhz can't divide it's speed to get exactly 115200 bps. Pairing it with another AVR using internal RC oscillator can complicate this situation even more - don't expect such configuration to work correctly especially at low voltages and temperatures without calibrating first.

Keep in mind that first ATmega devices (Atmega8, Atmega16 and so on) have separate factory calibrated oscillator values for different speeds - only the one for 1Mhz operation is loaded automatically. And it is said to be +/- 3% at 5V, 25°C.

Check for baudrate problems before blaming moo-cdc!

  • moo-cdc-20100130.zip - source code
  • moo-cdc-20100130-m8.hex - binary for Atmega8, LED at PC4
  • moo-cdc-20100130-m88.hex - binary for Atmega88, LED at PC4
